React-komposer

Github doc

  • 取代 createContainer & getMeteorData mixin
  • 组织 React container
  • 为组件从服务器获取数据
  • 不仅仅限于 Meteor，也支持 Rx.js/Redux/MobX.js

Mixin

  • 作用 - 组件之间 share 通用函数
  • 问题
    • implicit dependencies
    • name clashes
    • snowballing complexity
  • ES6 不支持 mixin
  • 已有太多 mixin 的 ES6 实现

React Container Pattern

  • 分离数据 data 和界面 UI
    • 数据 data => container
    • 界面 UI => UI 组件
  • Relay · Redux · Meteor

为什么使用 react-komposer

  • 尽量少使用 state
  • 使用 props 传递 data 和 action 到 UI 组件
  • 作为 container，为 UI 组件从服务器获取数据

React-komposer 功能

  • 数据获取（sub or produce）是主要功能
  • Loading 提示 (built-in)
  • 数据一旦到达，传递到 UI 组件
  • 错误提示
  • 重新获取或 sub 新数据
  • cleanup（unsub）

Meteor

composeWithTracker 替代 compose 

import {composeWithTracker} from 'react-komposer';
import PostList from '../components/post_list.jsx';

// reactive computation
function composer(props, onData) {
  if (Meteor.subscribe('posts').ready()) {
    const posts = Posts.find({}, {sort: {_id: 1}}).fetch();
    onData(null, {posts});
  };
};

export default composeWithTracker(composer)(PostList);
自动 unsubscribe
